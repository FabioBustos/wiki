# Revisión de Requisitos

Este documento describe el proceso de revisión de requisitos en el proyecto.

## Definición

La **Revisión de Requisitos** es un proceso sistemático para evaluar la completitud, consistencia, claridad y viabilidad de los requisitos de un sistema o software. Su objetivo es identificar y corregir problemas en las etapas tempranas del ciclo de desarrollo.

## Proceso

1.  **Preparación**: Distribuir los documentos de requisitos a los revisores.
2.  **Reunión de Revisión**: Discutir los requisitos, identificar ambigüedades, inconsistencias o faltantes.
3.  **Documentación de Hallazgos**: Registrar todos los comentarios, preguntas y defectos encontrados.
4.  **Resolución**: El equipo de producto y desarrollo trabaja para resolver los hallazgos.
5.  **Aprobación**: Los requisitos revisados son aprobados por los stakeholders relevantes.
