# Plantillas y Scripts

Este documento describe el uso de plantillas y scripts para mejorar la eficiencia y la consistencia en el proyecto.

## Definición

-   **Plantillas**: Son estructuras predefinidas o formatos estandarizados que sirven como punto de partida para la creación de nuevos documentos, módulos de código, componentes, etc. Ayudan a mantener la consistencia y aseguran que se incluyan todos los elementos necesarios.
-   **Scripts**: Son programas pequeños y automatizados diseñados para realizar tareas específicas de forma repetitiva o para automatizar procesos manuales. Mejoran la eficiencia, reducen errores y liberan tiempo para tareas más complejas.

## Importancia en el Proyecto

-   **Consistencia**: Aseguran que el código, la documentación y otros artefactos sigan un estilo y una estructura uniformes.
-   **Eficiencia**: Aceleran el proceso de desarrollo al proporcionar puntos de partida listos para usar y automatizar tareas repetitivas.
-   **Reducción de Errores**: Minimizan los errores humanos asociados con la creación manual y repetitiva de elementos.
-   **Mejores Prácticas**: Encapsulan y promueven las mejores prácticas de desarrollo y documentación.
-   **Onboarding**: Facilitan la incorporación de nuevos miembros al proporcionar herramientas para generar código y documentos rápidamente.

## Ejemplos de Uso

### Plantillas

-   **Plantillas de Documentación**: Como las utilizadas en esta wiki para crear nuevos documentos con frontmatter y estructura predefinida.
-   **Plantillas de Módulos NestJS**: Para generar la estructura básica de un nuevo módulo (controlador, servicio, módulo, etc.).
-   **Plantillas de Componentes React/Next.js**: Para crear nuevos componentes con una estructura estándar (archivos `.tsx`, `.module.css`, `.test.tsx`).
-   **Plantillas de Commits Git**: Para asegurar mensajes de commit consistentes y descriptivos.

### Scripts

-   **Scripts de Generación de Módulos/Componentes**: Como el `nestjs-module-generator` que automatiza la creación de la estructura de un módulo.
-   **Scripts de Despliegue**: Para automatizar el proceso de build, test y despliegue de la aplicación en entornos como [[Railway]] o [[Seenode]].
-   **Scripts de Formateo y Linting**: Para aplicar automáticamente reglas de estilo y calidad de código.
-   **Scripts de Migración de Base de Datos**: Para gestionar cambios en el esquema de la base de datos.
-   **Scripts de Limpieza**: Para eliminar archivos temporales o directorios de build.

## Herramientas que Utilizan Plantillas y Scripts

-   **[[Skills]]**: Muchas skills incluyen plantillas y scripts como parte de sus recursos para guiar y automatizar tareas.
-   **Generadores de Código**: Herramientas como `Nest CLI` o `Next.js CLI` utilizan plantillas internas para generar código.
-   **CI/CD Pipelines**: Los scripts son el corazón de los pipelines de integración y despliegue continuo.
